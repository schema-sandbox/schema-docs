# User-Visible Error Catalog

This catalog keeps public-preview errors understandable. Each user-visible error should answer three questions:

- **What happened:** the plain-language event.
- **Why it matters:** the product or safety consequence.
- **What to do next:** the smallest useful recovery step.

## Catalog Registry

| Code | What happened | Why it matters | What to do next | Severity |
|------|---------------|----------------|-----------------|----------|
| `path_outside_workspace` | The requested path is outside the selected workspace. | Schema Docs refused the operation to keep local files inside the workspace boundary. | Choose a file under the workspace or open the correct workspace first. | High |
| `unsupported_file_type` | The file type is not supported by the local intake pipeline. | The document cannot be converted into reviewed Markdown or AI context yet. | Save or export the file as DOCX, PDF, XLSX, CSV, MD, or TXT, then import again. | Medium |
| `optional_adapter_required` | A legacy Office format needs an optional system adapter. | The zero-dependency core does not parse that format directly. | Convert the file to DOCX/XLSX/PDF, or install the required adapter shown by `npm run doctor`. | Medium |
| `document_corrupt` | The Word document could not be opened as a valid DOCX package. | Extraction may be empty or misleading, so the file should not be sent to AI as trusted context. | Open it in Word/WPS, save a fresh DOCX copy, and import that copy. | High |
| `pdf_text_layer_missing` | The PDF has no readable text layer or behaves like a scanned document. | Schema Docs cannot reliably show what AI will see without OCR support. | Run OCR first or use a text-layer PDF before creating AI context. | High |
| `query_unsupported` | The local SQL query uses unsupported syntax. | The table must be filtered locally before AI review, and unsupported SQL cannot produce a safe filtered context. | Use SELECT with one WHERE comparison, LIKE, GROUP BY, ORDER BY, LIMIT, aggregates, or simple INNER JOIN syntax. | Medium |
| `query_table_not_found` | The selected table name was not found in the workspace. | The AI table context would be empty or point to the wrong dataset. | Run `tables <workspace>` or use the table picker, then retry with a listed table name. | Medium |
| `query_column_not_found` | A selected, filtered, grouped, or sorted column was not found. | The local filtered AI context would not match the intended table fields. | Copy a column name from `tables <workspace>` or the table picker, then retry the query. | Medium |
| `query_no_columns` | The SQL query did not select any columns. | Schema Docs cannot build a reviewed table preview without selected output fields. | Add at least one selected column or use `SELECT *` for a quick local preview. | Medium |
| `ai_confirmation_required` | The send action was requested without explicit confirmation. | Schema Docs never sends content to an AI endpoint until the user confirms the reviewed payload. | Review AI Will See, then check confirmation and send again. | High |
| `ai_handoff_context_empty` | The handoff bundle was requested without staged context. | An empty handoff bundle cannot prove what AI should receive or return. | Load a record, package, chunk, range, or pasted context before saving the handoff bundle. | Medium |
| `ai_send_gate_review_required` | Send Gate found sensitive data, credentials, or review-required content and blocked the send. | The content may leak secrets or low-confidence extraction into an AI model. | Mask or remove the flagged content, regenerate AI Will See, and retry only after the gate allows it. | High |
| `ai_override_reason_required` | A manual quality override was requested without an audit reason. | Overrides need a written reason so the exchange package and evidence trail explain the decision. | Add a short override reason, then retry the confirmed send. | Medium |
| `api_key_required` | No API key was provided for the confirmed AI send. | Schema Docs does not store API keys in profiles, so the request cannot be made without a session key. | Paste the API key for this local session and send again. | Medium |
| `api_prompt_empty` | The confirmed API request has no prompt content. | Empty prompts create misleading audit records and provider errors. | Load reviewed AI Will See content before sending to the configured API. | Medium |
| `api_request_failed` | The configured AI API returned an error response. | The reviewed context was not successfully delivered to the selected provider. | Check the API base URL, model, key, and provider status, then retry from the same reviewed context. | Medium |
| `api_response_non_json` | The configured AI API returned a response that was not JSON. | Schema Docs cannot verify the provider result or write a trustworthy AI response record. | Check that the API base URL points to an OpenAI-compatible JSON endpoint, then retry from the same reviewed context. | Medium |
| `runtime_unavailable` | The local JavaScript runtime or API bridge is not reachable. | The UI cannot import, preview, or package documents without the local API. | Run `npm run doctor`, check Desktop diagnostics, or start the local server with `npm run serve`. | High |
| `exchange_package_hash_mismatch` | Exchange package verification found a hash mismatch. | The package may have been modified after it was created. | Generate a fresh package and share the new receiver/trust report. | High |
| `exchange_package_unsafe_raw_file` | The exchange package contains raw source files that are not allowed. | Raw files can bypass the reviewed Markdown and Send Gate evidence path. | Rebuild the package from the prepared record instead of attaching raw source files. | High |
| `csv_empty` | The CSV dataset has no headers or data. | An empty dataset cannot be queried or converted into AI-ready Markdown. | Verify that the source CSV is not empty and has valid comma-separated values before importing. | High |
| `api_base_url_required` | The API profile is missing the base URL. | The local API client cannot route requests without a target endpoint URL. | Fill in a valid API base URL (e.g. `https://api.openai.com/v1`) in the settings panel. | Medium |
| `api_model_required` | The API profile is missing the model name. | The model request will be rejected by the endpoint if no model identifier is specified. | Provide the target model identifier (e.g. `gpt-4o`) in the settings panel. | Medium |
| `api_profile_name_required` | The API profile is missing a name. | API profiles must be named so they can be saved, listed, and selected in the workspace. | Enter a unique name for the API profile in the profile settings. | Medium |
| `record_not_found` | The requested record identifier does not exist in the workspace database. | Operations like previewing or exporting require a valid record reference to locate the workspace files. | Choose an existing record from the dashboard inbox or import a new file. | High |
| `record_markdown_missing` | The prepared record has no extracted Markdown content. | Schema Docs cannot generate AI context or exchange packages without normalized Markdown. | Click "Extract to Markdown" on the document first to populate the Markdown working copy. | High |
| `dataset_not_found` | The requested dataset identifier does not exist in the workspace database. | Local SQL queries cannot be executed against non-existent datasets. | Check that the dataset was imported successfully and select it from the workspace dashboard. | High |
| `document_not_found` | The requested document identifier does not exist in the workspace database. | The intake pipeline cannot extract text or export a document that cannot be found. | Choose a valid document from the inbox or import the source file again. | High |
| `ai_content_empty` | The AI context payload has no content. | The local safety check and external send cannot run on an empty document or empty selection. | Paste or load content into the AI context field first. | Medium |
| `ai_operation_unsupported` | The requested AI operation is not supported by the preview/send action. | Schema Docs only supports safe, audited prompt patterns (summarize, translate, extract, explain table) to prevent prompts from bypassing the Send Gate. | Select a supported operation from the dashboard or CLI options. | Medium |
| `ai_result_empty` | No AI response was found when writing back the results. | The workspace copy cannot record the AI outcome without a successful model response. | Run the AI send workflow first and confirm the model responded. | Medium |
| `api_profile_not_found` | The selected API profile could not be found in the workspace settings. | Requests cannot be sent to an AI provider without a valid, configured profile. | Select a different API profile or create a new profile in settings. | Medium |
| `dataset_importer_not_found` | No matching importer was found for the CSV or XLSX file copy. | The table dataset cannot be normalized or queried without a valid intake parser. | Export the file as CSV or XLSX and try importing it again. | High |
| `document_converter_not_found` | The document extraction converter was not found or is not supported. | The system cannot extract plain text or Markdown from this document type. | Convert the file to DOCX/PDF or install the optional system adapter using doctor. | High |
| `workspace_required` | No active workspace has been selected or opened. | Working copy files, manifest indexing, and local DB features require a valid workspace directory. | Open or create a local workspace from the dashboard folder picker or CLI. | High |
