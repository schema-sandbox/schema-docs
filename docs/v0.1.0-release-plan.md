# v0.1.0 Release Plan

## Release Readiness Judgment

Current core status is strong enough for a v0.1.0 public preview candidate:

- CLI, local API, SDK, smoke, Web UI checks, release checks, and tests are green.
- Runtime dependencies remain at 0.
- Markdown / DOCX / PDF exchange works through a Markdown exchange layer while Office/PDF files remain the visible user entry.
- Directory exchange packages can be generated, read back, and hash-verified.
- AI Send Gate has preview, confirmation, audit, evidence, and hard blocking for `review_required` content.
- F-012 desktop verification remains open until the final rebuilt Windows installer is visibly exercised and its evidence is closed against the exact release artifact hashes.

The remaining work is release-candidate discipline plus final F-012 closure: keep the candidate stable, rerun final gates, visibly verify the exact rebuilt installer, preserve real-sample confidence, and make the public-facing positioning clear.

### Expected Timing & Strategy
The release train focuses on weekly public-preview cycles. Structured DOCX and styled printable PDF exports are stabilized and verified; pixel-perfect reconstruction of arbitrary source layouts remains out of scope. Framing targets local-first AI document exchange workflows with safety constraints.

```bash
npm run desktop-preflight-check -- <preflight-dir>
npm run desktop-fixture-close -- --record <filled-record.json> --write
npm run desktop:dev
npm run desktop:build
```

* **`release-check`**: Confirms release documents, zero dependency posture, required scripts, served Web UI smoke, release artifact manifest, desktop verification fill/check/fixture-close, Tauri configuration, and packaged app/workflow smoke coverage.
* **`release-readiness`**: Integrates automatic release checks and strict fixture checks. It reports `readyForPublicTag: true` once F-012 passes and is closed.
* **`language-boundary-check`**: Enforces that default runtime/release files are English-only, allowing multilingual characters in allowed files, blocking mojibake everywhere.
* **`desktop-verification-*`**: Handles recording, filling, and verifying step-by-step F-012 manual testing evidence (WebView2, Node, diagnostics, first workflow, workspace/file pickers, and smoke runs).
* **`public-preview-package`**: Verifies the public preview installer handoff, recommended audience, primary NSIS installer, and release readiness summary.
* **`desktop-release-preflight` & `desktop-preflight-check`**: Generates and verifies portable preflight evidence packages before manual test handoffs, including the public preview package handoff command.
* **`desktop-fixture-close`**: Closes the F-012 fixture using the completed manual verification record.
* **`fixture-check --strict`**: The strict validation gate for sample workflows.
* **`ui-check` & `web-ui-smoke`**: Verify served layout HTML, JS module imports, DOM IDs, and local API connectivity.

6. Keep `npm run rc-check -- --json` and `npm run public-preview-package -- --json` green before every candidate build.
